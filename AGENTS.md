# Company-Wide AI Agent Guidelines

## Overview
This file defines universal guidelines for all AI coding agents used across the organization.

## General Principles
- Plan first

### Code Quality
- Write clean, self-documenting code. Prefer clarity over cleverness.
- Keep components and functions small and focused (ideally < 100 lines).
- Follow existing code conventions and patterns in each repository
- Maintain consistency with the surrounding codebase
- Write clean, readable, and maintainable code

### Security Best Practices
- Never commit secrets, API keys, or credentials to version control
- Use environment variables or secure secret management for sensitive data

### Documentation
- Update relevant documentation when making significant changes
- Add code comments only when necessary for complex logic

### Git Workflow
- Create feature branches for new work
- Use clear, descriptive commit messages
- Submit pull requests for code review
- Never force push to main/master branches
